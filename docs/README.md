# Project Documentation

This document provides an overview of the project directory structure and its documentation.

## Directory Structure

| Folder | Description |
| :--- | :--- |
| `docs` | Project documentation. |
| `docs/reports` | Project reports and formal documentation. |
| `docs/research` | Research notes, diagrams, and supporting materials. |
| `docs/weekly_updates` | Weekly progress logs and updates. |
| `storage` | Data and artifact storage. |
| `storage/jars` | Compiled JAR files (e.g., for data processing engines). |
| `storage/data` | Raw or processed datasets. |
| `storage/schemas` | Definitions of data structures (e.g., JSON schemas, Avro). |
| `cache` | Temporary files and performance caches. |
| `cache/checkpoints` | Checkpoints for fault-tolerant data pipelines (e.g., Flink). |
| `notebooks` | Jupyter notebooks for data analysis or experiments. |
| `src` | Core source code. |
| `src/producers` | Components responsible for generating or ingesting data. |
| `src/api` | Web API components. |
| `src/jobs` | Main data processing jobs/pipelines. |
| `infra` | Infrastructure and deployment configurations. |
| `infra/docker` | Docker configuration and Dockerfiles. |
| `infra/grafana` | Dashboards and configuration for Grafana monitoring. |
| `infra/config` | General configuration files for the infrastructure. |
| `tests` | Project test suite. |
